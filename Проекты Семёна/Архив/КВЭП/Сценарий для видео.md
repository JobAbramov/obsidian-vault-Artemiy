## Начало
Добрый день! Процесс импорта, который задействует Гугл Таблицы и виджет Триггеры, требует предварительной настройки, а именно: 
- Адаптация таблицы под определённый формат;
- Настройка дополнения для Гугл Таблиц для отправки веб-хуков;
- Настройка приёмки веб-хуков на стороне Триггеров;
## Формат данных таблицы
Для того, чтобы обеспечить корректность переносимых данных, необходимо, чтобы таблица имела определённый формат:
- Для каждого столбца должен быть один тип данных: в столбце с номером телефона должны содержаться только номера телефона, столбец с ссылками на веб-сайт содержит ссылки на веб-сайт формата https://<название-сайта>. Если столбец обладает сложно структурой, например поле "Реквизиты" или "Юр.лицо", то необходимо каждый пункт вынести в отдельный столбец;
- Шапка таблицы должна содержать подписи для всех столбцов;
- Ссылки должны содержать быть в формате https://<название-сайта>. Не допускается, если ссылка "зашита" в текст;
- Необходимо, чтобы у таблицы было одно ключевое поле, по которому можно однозначно определить сущность/строку;
- Ключевое поле/столбец должно содержать только одно значение. Список из нескольких значений в ключевом поле не допускается;
- Количество ключевых полей в одной строчке может определяться количеством сущностей, которые отразить в системе, как значимые: например, в таблице в одной могут содержаться реквизиты как компании, так и контактного лица. Если требуется находить дубли по компании или/и по контакту, требуется определить ключевые поля как контакта, так и компании. Для компании ключевым полем может служить ИНН, а для контакта - Телефон или почта;

## Нормализация данных
Если таблица не приведена в нужный формат, то привести в нужный формат можно стандартными средствами, как например через формулы, форматирование ячеек, копирование/вставка в Гугл Таблицах. Если данных слишком много и требуется более сложная обработка, можно прибегнуть к расширениям, добавляющие нейросети для обработки таблиц или же к макросам.

## Подключение расширения для отправки веб-хуков
Чтобы отправлять веб-хуки из Гугл таблиц, требуется подключить расширение Trigger & Send. Делается это следующим образом: Расширение -> Дополнения -> Установить дополнения. Далее ищем наше расширение и устанавливаем. Теперь мы можем отправлять веб-хуки.

Для того, чтобы открыть вкладку с настройками дополнения, необходимо сделать так: Расширения -> Trigger & Send -> Configure. Справа от таблицы появится меню, в котором имеются следующие настройки: 
 1. Trigger Data - тип события, после которого отправляется хук:
	 - Now - хук отправится сразу после нажатия на кнопку "Post". Одномоментно можно отправить не более 500 хуков;
	 - Schedule - хуки будут отправляться ежечасно по 200 строк-хуков за час;
	 - On Form Submit -> хук будет отправляться после того, как Гугл Формы отправят данные в таблицу. В хуке будут содержаться данные из формы;
	 - On Event Change -> хук будет отправляться после того, как изменится таблица и будет содержать строчку, в которой произошло изменение.
2. Send data - ссылка, на которую будут отправляться хуки.
3. Select Range - режим выборки строк для обработки:
	- All Columns - будет выгружаться вся таблица;
	- Manual Range - выбор произвольного диапазона столбцов и строк
4. Row Number to start - порядковый номер строки, с которой начнётся отправка хуков. Данное поле появляется при опции All Columns;
5. Range in A1 notation - диапазон ячеек, данные из который будут отправляться в хуке. Поле появляется при опции Manual Range;

## Настройка приёма хуков на стороне Триггеров
Теперь нам необходимо настроить сценарии в Триггерах, которые смогут обработать входящий хук.

Для этого необходимо подготовить приёмщик хуков в Триггерах. Полученную ссылку мы вставляем в поле Send Data

Затем отправляем тестовый хук. После того, как он пришёл, мы можем настроить сопоставление полей для поиска нужной сущности (компании или контакта). Для этого мы нажимаем на название параметра хука и вставляем в поле для сопоставления. Затем выбираем сценарий, который должен сработать при найденной компании.

Также мы настраиваем сопоставление полей и сценарии, если компании нет в системе

В самих сценариях мы так же можем использовать параметры входящего хука
